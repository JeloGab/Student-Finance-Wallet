### 4. CORS Fix — DONE
All browser requests were blocked with "No content available for preflight request".

**Root cause:** CORS config didn't explicitly allow the `Authorization` header — browsers send a preflight OPTIONS request that was failing.

**Fix in `server/index.js`:**
```js
app.use(cors({
  origin: process.env.CLIENT_URL || 'http://localhost:5173',
  methods: ['GET', 'POST', 'PUT', 'PATCH', 'DELETE', 'OPTIONS'],
  allowedHeaders: ['Content-Type', 'Authorization'],
  credentials: true
}))
```

Also removed `app.options('*', cors())` — Express 5 doesn't support bare `*` in routes.

### 7. Bug Fixed — `logged_at` vs `created_at` column mismatch
**Root cause:** Used `replace_all` to fix `created_at → logged_at` in `adminController.js` which incorrectly replaced `payment_transactions` references too. Only `activity_logs` uses `logged_at`. `payment_transactions` uses `created_at`.

**Lesson:** Never use `replace_all` across mixed table references in the same file.

**Files fixed:**
- `server/src/controllers/adminController.js` — restored `created_at` for `payment_transactions`
- `server/src/controllers/paymentController.js` — added `logged_at: new Date().toISOString()` to activity_logs insert

**Confirmed column names:**
| Table | Timestamp column |
|---|---|
| `payment_transactions` | `created_at` |
| `activity_logs` | `logged_at` |
| `student_accounts` | `updated_at` |

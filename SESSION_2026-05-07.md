## Decisions Made

### CSS uses pull model
CSS does not receive data from SFW. CSS calls ESB → ESB calls SFW → SFW returns status. SFW does not push anything outward.

### CSS route has no auth
Open route — professor controls who can call it via ESB. Can add token validation later if required.

### SRM integration uses stub mode
If `ESB_JWT_TOKEN` is missing, SRM check is skipped silently. In production (token set), real call happens. Stub is kept as a safety net in case token expires during demo.

### Auto-insert after SRM verification
SRM is the gate — student accounts are only auto-created if SRM confirms the student_id is valid. Prevents fake accounts from being created with random IDs.

### student_name and program are nullable
Confirmed in Supabase — no ALTER TABLE needed. Will be updated once SRM response shape is known.

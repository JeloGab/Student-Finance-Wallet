## What Was Accomplished Today

### 1. Database — Supabase Setup
- Dropped Prisma entirely — using `@supabase/supabase-js` directly for all DB operations
- Created 4 tables manually via Supabase SQL Editor:
  - `student_accounts` — core financial account per student (student_id, student_name, program, total_due, total_paid, status, updated_at)
  - `payment_transactions` — individual payment records (amount, reference_no, payment_date, payment_method, recorded_by, status)
  - `fee_records` — fee breakdown per enrollment (tuition, lab_fees, other_charges, adjustments)
  - `activity_logs` — audit trail of all actions (action, performed_by, details JSONB)
- RLS was enabled on all tables — server uses service role key which bypasses RLS

### 2. Server — Express Backend (`/server`)
Built from scratch with this structure:
```
server/
├── index.js                        Express entry point, CORS, routes wired
├── src/
│   ├── config/supabase.js          Supabase service role client (bypasses RLS)
│   ├── middleware/
│   │   ├── auth.js                 Verifies Supabase JWT, extracts user + role
│   │   └── validate.js             Zod validation middleware
│   ├── routes/
│   │   ├── payments.js             POST /, GET /recent, GET /today-total
│   │   └── students.js             GET /:studentId
│   ├── controllers/
│   │   ├── paymentController.js    recordPayment, getRecentPayments, getTodayTotal
│   │   └── studentController.js    lookupStudent
│   └── validators/
│       └── paymentValidator.js     Zod schema for payment input
```

**Key server behaviors:**
- All routes require a valid Supabase JWT in `Authorization: Bearer <token>` header
- All payment routes require `role === 'finance_staff'` in user metadata
- Payment recording: checks student exists → checks duplicate reference_no → inserts transaction → updates total_paid → recalculates status → logs activity
- Status rule: `CLEARED` only if `total_due > 0` AND `total_paid >= total_due`, otherwise `ON_HOLD`
- `z.coerce.number()` used for amount so string inputs from forms are handled

**Server .env variables needed:**
```
SUPABASE_URL=
SUPABASE_ANON_KEY=
SUPABASE_SERVICE_KEY=
CLIENT_URL=http://localhost:5173
PORT=3000
```

**Run server:** `npm run dev` inside `/server`

### 4. Authentication
- Uses Supabase Auth (email + password)
- Role stored in `user_metadata.role` at registration (`finance_staff` or `student`)
- Login redirects `finance_staff` → `/finance/payments`, `student` → `/dashboard`
- `Layout.jsx` guards all protected routes — redirects to `/login` if no session
- Server verifies JWT on every API call using `supabase.auth.getUser(token)`

### 5. Payment Management Page
- **Student ID lookup:** triggers on Enter or blur from the field — calls `GET /api/students/:id` — shows name + program if found, error if not
- **Form fields:** Student ID, Amount (PHP), Date, Reference Number, Payment Method (GCash / Maya / Online Bank / Manual Payment)
- **Submit:** calls `POST /api/payments` — shows success with new student status or error message
- **Ledger:** fetches last 10 payments on mount and after each successful submission
- **Today's total:** fetches from `GET /api/payments/today-total` on mount

### 6. UI Design System
- Font: **Manrope** (Google Fonts)
- Icons: **Material Symbols Outlined** (Google Fonts)
- Framework: **Tailwind CSS v3** with custom theme
- Color palette, spacing, and font sizes all extracted from the Figma-exported HTML and defined in `tailwind.config.js`
- To add a new page: create the page file, import into `App.jsx`, wrap with `<Layout>`, add nav item to `Sidebar.jsx`
